knowra 本地启动指南

本页基于根目录 README.md、backend/README.md、front/README.md、 docker-compose.yml 和项目脚本整理，用于快速启动 knowra 的数据库、后端和前端。

后端：FastAPI + uv 前端：Vue 3 + Vite 数据库：PostgreSQL + pgvector 默认前端端口：5173 默认后端端口：8000

启动顺序

  1. 先启动 PostgreSQL，因为后端迁移和服务依赖数据库。
  2. 再初始化并启动后端，后端默认服务地址是 http://localhost:8000。
  3. 最后初始化并启动前端，前端默认服务地址是 http://localhost:5173。
前端开发环境通过 Vite proxy 把 /api 转发到后端，所以前端业务代码不需要写死 localhost:8000。

环境要求

模块 要求 来源
后端 Python 3.14.5、uv README.md、backend/pyproject.toml
前端 Node.js 24.15.0、npm 11+ front/package.json、front/README.md
数据库 Docker Compose v2 或兼容工具 README.md、docker-compose.yml

1. 启动数据库

在项目根目录执行：

docker compose up -d postgres

默认数据库连接串：

postgresql+psycopg://knowra:knowra@localhost:5432/knowra
如果本机没有 docker compose 子命令，需要先安装 Docker Compose v2，或使用兼容工具启动 docker-compose.yml 中的 postgres 服务。

2. 启动后端

打开一个终端，在项目根目录执行：

cd backend
uv sync
cp .env.example .env
uv run alembic upgrade head
uv run uvicorn app.main:app --reload
服务地址 http://localhost:8000
API 前缀 /api

后端环境文件

backend/.env.example 中的默认关键配置如下：

API_PREFIX=/api
BACKEND_CORS_ORIGINS=http://localhost:5173
DATABASE_URL=postgresql+psycopg://knowra:knowra@localhost:5432/knowra

3. 启动前端

再打开一个终端，在项目根目录执行：

cd front
npm install
cp .env.example .env
npm run dev
访问地址 http://localhost:5173
API 基础路径 VITE_API_BASE_URL=/api
当前前端会通过 GET /api/users/me 读取当前用户上下文；该阶段不包含登录、注册、登出和用户切换。

快速排查

  • 后端启动或迁移失败时，先确认 postgres 容器已启动，并且本地 5432 端口没有被其他服务占用。
  • 前端页面请求失败时，确认后端运行在 localhost:8000，并且前端环境变量仍为 VITE_API_BASE_URL=/api。
  • 依赖安装失败时，先确认 Python、Node.js、npm 版本符合项目要求。
