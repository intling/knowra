# knowra 项目章程

## 1. 项目定位与目标
**knowra** 是一个私有化部署的个人 AI 知识助手，面向个人用户与团队用户，帮助他们将自己的课程讲义、论文、笔记、项目文档等资料上传到系统，构建可检索的知识库。

本项目目标是搭建“知寻”这样一个私人 AI 助手：
- 能理解多种文档格式并自动解析内容；
- 能基于语义检索找到最相关的知识片段；
- 能在用户提问时生成带引用来源的回答；
- 能在私有环境中运行，确保数据不出本地。

## 2. 核心功能范围
### 2.1 重点支持内容
- 文档理解：支持 Markdown、PDF、TXT 等常见格式，自动解析文档并进行分块、向量化、语义索引构建。
- 语义检索：非关键词匹配，而是理解用户问题含义，检索最相关内容。
- 查询优化：内置 Query Rewrite 和 HyDE 机制，提升模糊或低质量查询的检索召回率。
- RAG 对话：将检索到的文档片段作为上下文，辅助大语言模型生成回答，并提供真实引用来源。
- 知识图谱融合：从文档中抽取实体与关系，构建概念图谱，支持跨文档关联推理。
- 自评估实验平台：内置检索质量评测体系，支持不同分块策略、Embedding 模型、Top-K 配置的对比实验。
- 私有部署：所有数据和模型运行在用户本地或用户控制的私有环境中，不依赖外部数据泄露风险。

### 2.2 当前阶段重点
- 先构建系统基础架构：前后端联调、数据库连接、基本 API、健康检查、文档上传与解析接口框架。
- 逐步补齐语义检索与向量索引能力，建立可扩展的检索服务接口。
- 为后续 RAG、Query Rewrite、知识图谱、评估平台留出清晰扩展点。

### 2.3 暂不覆盖内容
- 完整的生产级用户认证与权限管理；
- 复杂的多租户运维与云端自动扩缩容；
- 业务线级完整文件管理、共享与协作功能；
- 智能推荐、自动摘要、自动问答外部模型托管等高级服务。

## 3. 关键成功指标
1. 系统可启动，后端服务健康接口可用，数据库与 pgvector 正常连接。
2. 前端页面加载成功，能通过代理调用后端接口并展示结果。
3. 文档解析与语义索引流程有初步实现框架，可为后续 RAG 和检索能力打基础。
4. 项目具备私有部署思路，敏感信息配置和本地运行方案清晰。

## 4. 技术架构标准
### 4.1 核心技术栈
- **后端**：Python 3.14 + FastAPI + SQLModel + PostgreSQL + pgvector
- **前端**：Vue 3 + TypeScript + Vite
- **数据库与检索**：PostgreSQL + pgvector，用于存储元数据和向量索引
- **工程工具**：`pytest`/`vitest`、`ruff`、`eslint`

### 4.2 设计与扩展原则
- **模块化**：文档解析、向量化、检索、RAG、知识图谱各模块应清晰分层，便于替换和升级。
- **私有部署优先**：所有关键数据处理与模型调用应支持本地执行，避免默认开放云端依赖。
- **接口统一**：后端 API 统一使用 `/api` 前缀，前端通过 Vite Proxy 调用后端接口。
- **配置隔离**：敏感信息不得硬编码，统一使用 `.env` 和配置模板管理。
