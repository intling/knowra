# knowra 后端章程

## 1. 项目定位与目标
**knowra 后端** 是 knowra 项目的后端部分，基于 FastAPI + SQLModel 构建的 API 服务。
**当前阶段目标**：建立后端基础骨架，实现数据库连接与健康检查接口，确保后端服务可启动、可联调、可维护。

## 2. 范围界定
### 2.1 包含内容
- **服务框架**：FastAPI、Uvicorn、ASGI 应用结构。
- **数据库层**：PostgreSQL、pgvector 扩展、SQLModel ORM、Alembic 迁移。
- **接口**：统一 `/api` 前缀、健康检查接口、基础路由组织。
- **工程化**：Python 项目配置、环境变量管理、日志与错误处理、测试覆盖。

### 2.2 不包含内容
- **AI 相关**：文档解析、向量检索、LLM 调用。
- **业务相关**：用户认证、权限系统、文件存储。
- **生产运维**：生产级部署与 CI/CD 流水线（当前仅关注本地开发与联调）。

## 3. 关键成功指标
1. **后端可启动**：服务能够正常启动，应用无致命错误。
2. **健康检查通过**：访问 `/api/health` 返回 HTTP 200。
3. **数据库连接正常**：与 PostgreSQL 数据库建立连接，必要时能够执行迁移。

## 4. 技术架构标准
### 4.1 核心技术栈
- **语言**：Python 3.14.5
- **Web 框架**：FastAPI
- **ORM**：SQLModel
- **数据库**：PostgreSQL + pgvector
- **迁移**：Alembic
- **测试**：Pytest
- **配置管理**：环境变量 `.env` / `env_settings`
- **格式化与静态检查**：ruff

### 4.2 交互与通信规范
- **统一前缀**：所有后端接口必须以 `/api` 开头。
- **开发代理**：前端通过 Vite Proxy 转发至后端 `localhost:8000`，避免跨域问题。
- **配置管理**：敏感配置不硬编码，统一使用 `.env` 文件和 `backend/core/config.py` 管理。
